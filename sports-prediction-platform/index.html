🚀 Installation en 5 Étapes

1
Cloner le Repository
Téléchargez le code source du projet
cd /workspace
git clone [repository-url]
cd sports-prediction-platform
2
Configurer les Variables d'Environnement
Créez votre fichier .env avec vos clés API
⚠️ Note: Remplacez les valeurs XXX par vos propres clés API
# Copiez le fichier exemple
cp .env.example .env

# Éditez avec vos clés
nano .env
APIFootball
Clé pour données sportives
OpenWeatherMap
Données météorologiques
Database URL
Connexion PostgreSQL
Redis URL
Configuration cache
3
Lancer l'Infrastructure avec Docker
Démarrez tous les services (PostgreSQL, Redis, RabbitMQ, Backend, Frontend)
# Démarrer tous les services
docker compose up -d

# Vérifier l'état des services
docker compose ps

# Voir les logs en temps réel
docker compose logs -f
4
Initialiser la Base de Données
Exécutez le schéma SQL pour créer les tables
# Se connecter à PostgreSQL
docker compose exec postgres psql -U admin -d sports_prediction

# Exécuter le schéma
\i /docs/database-schema.sql

# Ou depuis l'hôte
cat docs/database-schema.sql | docker compose exec -T postgres psql -U admin -d sports_prediction
5
Vérifier l'Installation
Accédez aux différents services
🌐 Dashboard Frontend
http://localhost:3000
⚡ API Backend
http://localhost:8000
📚 Documentation API
http://localhost:8000/docs
📊 Monitoring Grafana
http://localhost:3001
Ouvrir Swagger UI Ouvrir Dashboard

📖 Commandes Utiles

Gestion des Containers

# Arrêter tous les services
docker compose down

# Redémarrer un service spécifique
docker compose restart backend

# Reconstruire les images
docker compose build --no-cache

# Nettoyer les ressources inutilisées
docker system prune -a

Logs et Debugging

# Logs en temps réel
docker compose logs -f backend

# Logs d'un service spécifique
docker compose logs -f postgres

# Accéder au shell du backend
docker compose exec backend bash

# Accéder à la base de données
docker compose exec postgres psql -U admin -d sports_prediction

Tests et Validation

# Lancer les tests backend
docker compose exec backend pytest

# Vérifier la santé des services
curl http://localhost:8000/health

# Tester une prédiction
curl -X POST http://localhost:8000/api/v1/predictions \
  -H "Content-Type: application/json" \
  -d '{"match_id": 12345}'

🔧 Dépannage

Problème: Ports déjà utilisés
Solution: Modifiez les ports dans docker-compose.yml ou arrêtez les services conflictuels.
Problème: Base de données inaccessible
Solution: Vérifiez que PostgreSQL est démarré avec docker compose ps et attendez 30 secondes après le premier démarrage.
Problème: Erreurs de dépendances Python
Solution: Rebuild le container backend: docker compose build backend

Vérification de Santé

# Vérifier tous les services
docker compose ps

# Test de connectivité PostgreSQL
docker compose exec postgres pg_isready -U admin

# Test de connectivité Redis
docker compose exec redis redis-cli ping

# Vérifier l'espace disque
df -h
